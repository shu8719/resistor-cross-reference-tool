# Thin Film Resistor Cross-Reference Tool

## 概要

電子部品メーカー4社（KOA・YAGEO・Vishay・Panasonic）の薄膜チップ抵抗品番を入力すると、RG/RGVシリーズの代替候補品番を提案するWebアプリです。
逆方向（RG/RGV品番 → 他社相当品）の検索にも対応しています。
業務で発生していた手作業の品番確認を効率化するために開発した個人開発プロジェクトです。

## できること

* **単体解析** — 品番を1件入力すると、サイズ・抵抗値・公差・TCR（温度係数）を自動で抽出し、代替候補品番を提案します
* **Excel一括解析** — Excelファイルをアップロードすると、全行を一括処理し、結果を追加したExcelファイルをダウンロードできます
* **逆引き** — RG/RGV品番を入力すると、KOA・YAGEO・Vishay・Panasonicの相当品番を一覧表示します
* **日本語 / English 表示切替** — 画面の表示言語をボタン1つで切り替えられます
* **判定理由の表示** — 代替候補品番を提案できない場合も、理由（例:「TCRレンジ非対応」）を返します

## 対応している品番

| メーカー       | シリーズ          | 入力例                 |
| ---------- | ------------- | ------------------- |
| KOA        | RN73R / RN73H | `RN73R1JTTD1002B25` |
| YAGEO      | RT            | `RT0402BRD071KL`    |
| Vishay     | TNPW          | `TNPW04021K00BEEA`  |
| Panasonic  | ERA（A-typeのみ） | `ERA3ARB1002V`      |
| RG/RGVシリーズ | RG / RGV      | `RG1608N-1002-B-T5` |

## 使い方

```bash
# 依存ライブラリをインストール
pip install -r requirements.txt

# サーバーを起動
python app.py
```

ブラウザで `http://localhost:5000` を開いてください。

`sample/sample_input.csv` に動作確認用の品番サンプルがあります。
Excel一括解析を試す場合は、`.csv` を `.xlsx` に変換してからアップロードしてください。

## 判定結果の意味

| 判定      | 意味                        |
| ------- | ------------------------- |
| ◎       | 代替候補品番の提案あり               |
| ！       | 対応範囲外（サイズ・TCR・抵抗値が非対応）    |
| ×       | 厚膜抵抗など、対象外の品種             |
| △       | 品番の形式が不明、またはコードが未定義       |
| REVERSE | RG/RGV品番の入力を検出（逆引きモードに遷移） |

## 技術スタック

| 分類        | 技術                            |
| --------- | ----------------------------- |
| バックエンド    | Python / Flask                |
| データ処理     | pandas / openpyxl             |
| フロントエンド   | HTML / CSS / JavaScript       |
| UIフレームワーク | Vue 3（CDN）/ Tailwind CSS（CDN） |
| テンプレート    | Jinja2（変数記法を `[[ ]]` に変更）     |

FlaskのテンプレートエンジンであるJinja2とVue 3は、どちらも `{{ }}` を変数展開に使います。
そのため、Jinja2側の区切り文字を `[[ ]]` に変更し、Vueと競合しないようにしました。

## プロジェクト構成

```text
cross-reference/
├── app.py          # アプリの起動・ルート登録
├── routes/         # URLとHTTPメソッドの定義（POST /analyze, POST /upload_bulk）
├── services/       # 品番解析・代替品番生成などのビジネスロジック
├── parsers/        # メーカーごとの品番パーサー（各社独立ファイル）
├── utils/          # 品番正規化・抵抗値コード変換などの共通関数
├── data/           # 仕様テーブルや定数の定義
└── templates/      # HTML（Vue 3を使ったUI）
```

処理の流れは `routes → services → parsers → utils` の一方向になるよう設計しています。

## ロジックの流れ

1. **品番の正規化** — 入力から記号・スペースを除去し、大文字に統一します
   例: `rn73r-1j` → `RN73R1J`

2. **メーカー・シリーズの判定** — 正規表現でKOA・YAGEO・Vishay・Panasonic・RG/RGVのいずれかを判定します

3. **スペックの抽出** — サイズ・抵抗値・公差・TCRを品番コードから読み取ります

4. **仕様範囲の確認** — 抽出したスペックがRG/RGVシリーズの対応範囲内かどうかを確認します

5. **代替候補品番の生成** — 対応範囲内であれば、シリーズの命名規則に従って代替候補品番を組み立てます

6. **結果を返す** — 範囲外・対象外・形式不明の場合も、理由を付けて結果を返します

## 工夫した点

### メーカーごとにパーサーを分けた

KOA・YAGEO・Vishay・Panasonic・RG/RGVのパーサーを、それぞれ独立したファイルに分けています。
新しいメーカーを追加する場合も、既存のコードへの影響を抑えやすい構成にしました。

### Excel一括処理に対応した

Excelファイルをアップロードすると、全行を一括処理し、判定結果を追加したExcelファイルを返します。
列名に「PN」や「PART」が含まれる列を品番列として自動認識するため、さまざまなExcel形式に対応しやすくしています。

### 提案できない場合も理由を返す

代替候補品番を提案できない場合でも、「範囲外（TCR非対応）」「厚膜抵抗・対象外」などの理由を返します。
単にエラーにするのではなく、ユーザーが次の判断をしやすいようにしました。

### FlaskとVue 3を組み合わせた軽量なUI

Node.jsやビルドツールを使わず、Vue 3をCDNで読み込む構成にしました。
Python / Flaskだけでバックエンドと画面を動かせるため、環境構築がシンプルです。
